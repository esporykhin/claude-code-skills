# amoCRM skill — конфигурация

## Как получить долгосрочный токен

1. В amoCRM: **Настройки → Интеграции → Создать интеграцию → Внутренняя интеграция**.
2. Заполни название и права (отметь нужные scope: сделки, контакты, компании и т.п.).
3. Открой созданную интеграцию → вкладка **"Ключи и доступы"** (или **"Долгосрочный токен"**).
4. Жми **"Сгенерировать долгосрочный токен"**, выбери срок (до нескольких лет).
5. Токен показывается один раз — скопируй сразу.

Также запиши:
- **Поддомен** аккаунта (для `mycompany.amocrm.ru` это `mycompany`).
- **Домен платформы** — `amocrm.ru` или `amocrm.com`.

## Установка токена

### Вариант 1 — дать агенту (он сам настроит)

Просто передай агенту в чате subdomain + токен. Он создаст `config/.env` с правильными значениями.

### Вариант 2 — руками

```bash
cp config/.env.example config/.env
# открой config/.env и замени YOUR_SUBDOMAIN, YOUR_LONG_LIVED_TOKEN
chmod 600 config/.env
```

## Альтернативы

- `AMOCRM_ACCESS_TOKEN=... AMOCRM_SUBDOMAIN=... ./scripts/amocrm-request.sh ...` — переменные окружения переопределяют `config/.env` на один вызов.
- Файл `config/.env` не попадает в git (см. `.gitignore`).

## Проверка

```bash
./scripts/amocrm-request.sh GET /api/v4/account
```

Должен вернуться JSON с данными твоего аккаунта.
